# RSDQL_GraphMatching 工程详解

## 一、项目目标 🎯

这个项目要解决的核心问题是：**如何把微服务（任务）高效地部署到云边计算集群（UAV无人机/服务器）上？**


---

## 二、核心概念 📚

### 1. 资源拓扑图 (Resource Graph)
- **是什么**：代表底层基础设施（UAV无人机群、服务器）
- **包含什么**：
  - 每个节点的CPU、内存大小
  - 节点之间的网络带宽、延迟

### 2. 任务拓扑图 (Task Graph)
- **是什么**：代表微服务之间的依赖关系
- **包含什么**：
  - 每个服务的CPU/内存需求
  - 服务之间的通信需求
- **例子**：A服务需要先调用B服务，B服务需要调用C服务

### 3. 图匹配 (Graph Matching)
- **是什么**：把任务图映射到资源图的过程
- **目标**：让部署后的效果最好（成功率高、延迟低）
- **方法**：匈牙利算法、贪心算法、启发式算法等

### 4. 反馈优化 (Feedback Optimization)
- **是什么**：把部署效果反馈给任务图生成器，优化任务图结构
- **反馈指标**：成功率、延迟、重调度次数

### 5. 强化学习 (Rainbow DQN)
- **是什么**：让AI自动学习最优的部署策略
- **Rainbow DQN**：DQN的升级版，结合了6种技术

## 四、各模块功能详解

### 1. dataSet/data.py 📂
```
作用：加载XML数据，生成UAV节点和服务
输出：
  - data.uav_nodes: UAV节点字典
  - data.service_nodes: 服务节点字典
  - data.uav_edges: UAV之间的边（带宽、延迟）
```

### 2. resource_graph.py 🖥️
```
作用：构建资源拓扑图
功能：
  - get_node_feature(): 获取节点特征（CPU、内存）
  - get_shortest_path_distance(): 计算节点间最短路径
  - get_all_node_features(): 获取所有节点特征
```

### 3. task_graph.py 📋
```
作用：构建任务拓扑图（支持资源感知生成）
功能：
  - get_service_feature(): 获取服务特征
  - generate_resource_aware_task_graph(): 生成资源感知任务图
  - adjust_topology(): 根据反馈调整任务图
```

### 4. graph_matcher.py 🔗
```
作用：将任务图映射到资源图
方法：
  - hungarian(): 匈牙利算法（最优匹配）
  - greedy(): 贪心算法（快速）
  - learned(): 学习型算法
  - heuristic(): 启发式算法
```

**例子**：
```python
from graph_matcher import GraphMatcher

matcher = GraphMatcher(resource_graph, task_graph)
mapping = matcher.match(method='hungarian')  # 执行匹配
print(f"匹配分数: {matcher.match_score}")
print(f"部署方案: {matcher.get_deployment_plan()}")
```

### 5. feedback_optimizer.py 🔄
```
作用：根据部署效果反馈，优化任务图
反馈指标：
  - R_success: 任务成功率
  - T_latency: 平均延迟
  - F_resched: 重调度次数
优化操作：
  - 拆分任务、合并任务
  - 调整任务优先级
  - 修改依赖关系
```

### 6. env.py 🎮
```
作用：强化学习环境（模拟部署场景）
方法：
  - reset(): 重置环境
  - step(action): 执行动作，返回(state, reward, done, info)
  - _calculate_reward(): 计算奖励
```

**例子**：
```python
from env import Env

env = Env('./dataSet/data.xml')
state, valid, message = env.reset()  # 初始化

# 执行动作（选择不同的匹配算法）
next_state, reward, done, info = env.step(action=1)

print(f"奖励: {reward}")
print(f"匹配分数: {info['match_score']}")
print(f"成本: {info['cost']}")
```

### 7. train.py 🚀
```
作用：训练强化学习智能体
流程：
  1. 加载数据
  2. 创建环境
  3. 创建智能体（SimpleAgent 或 RainbowDQNAgent）
  4. 预热（填充经验池）
  5. 训练循环
  6. 评估和保存模型
```

**运行训练**：
```bash
python train.py
```

### 8. rainbow_agent.py 🤖
```
作用：Rainbow DQN 智能体实现
包含技术：
  ✓ Dueling DQN    - 分离价值和优势
  ✓ Double DQN     - 减少Q值过估计
  ✓ Prioritized Replay - 优先回放重要样本
  ✓ NoisyNet       - 参数化噪声探索
  ✓ N-Step         - N步回报
```

```bash

```

### 方式3：自定义实验
```python
from env import Env
from dataSet.data import Data
from graph_matcher import GraphMatcher
from resource_graph import ResourceGraph
from task_graph import TaskGraph

# 1. 加载数据
data = Data('./dataSet/data.xml')

# 2. 创建资源图
resource_graph = ResourceGraph(data.uav_nodes, data.uav_edges)

# 3. 创建任务图
task_graph = TaskGraph(data.service_nodes, data.service_edges)

# 4. 创建匹配器并匹配
matcher = GraphMatcher(resource_graph, task_graph)
mapping = matcher.match(method='greedy')

# 5. 查看结果
print(f"匹配分数: {matcher.match_score}")
print(f"部署方案: {matcher.get_deployment_plan()}")
```

## 六、强化学习概念补充

### 什么是DQN？
- DQN（Deep Q-Network）：用深度神经网络来估计Q值（动作价值）
- Q值：表示在某个状态下执行某个动作的「好坏程度」

### 什么是Rainbow DQN？
是多种DQN改进技术的「集大成者」：

| 技术 | 作用 |
|------|------|
| Dueling DQN | 分别估计「状态价值」和「动作优势」 |
| Double DQN | 用两个网络减少Q值过估计 |
| Prioritized Replay | 优先学习「重要」的经验 |
| NoisyNet | 用噪声代替随机探索 |
| N-Step | 加速学习 |
| Distributional | 学习分布而不是单值 |

## 八、常见问题 ❓

### Q1: 为什么训练结果没有变化？
A: 可能原因：
1. 环境是「单步」环境，每个episode只有一步
2. 奖励函数设计不够敏感
3. 训练轮数不够

### Q2: 如何修改数据？
A: 编辑 `dataSet/data.xml`，格式如下：
```xml
<uav_node id="0">
    <cpu>100</cpu>
    <memory>128</memory>
</uav_node>
<service_node id="0">
    <cpu_demand>10</cpu_demand>
    <memory_demand>16</memory_demand>
</service_node>
```

### Q3: 如何调整超参数？
A: 在 `train.py` 或 `compare_agents.py` 中修改：
```python
LEARNING_RATE = 0.001    # 学习率
GAMMA = 0.9              # 折扣因子
MEMORY_SIZE = 10000      # 经验池大小
MAX_EPISODE = 1000       # 训练轮数
```
